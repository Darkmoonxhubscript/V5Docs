Getting Started

Initialize DarkLib V5 by loading the library through the following loadstring:

local DarkLib = loadstring(game:HttpGet("https://raw.githubusercontent.com/Darkmoonxhubscript/DarkLibV5/refs/heads/main/source.lua", true))()

Window Creation

Complete Window Setup

local Window = DarkLib:CreateWindow({
    Name = "Your Hub Name ",                    -- Main title displayed in the window header
    Subtitle = "Your subtitle here",            -- Secondary text shown below the main title
    LogoID = "10734897102",                    -- Roblox asset ID for the logo image
    Size = UDim2.new(0.95, 0, 0.8, 0),        -- Window dimensions (width, height as percentages)
    LoadingEnabled = true,                      -- Whether to show loading screen on startup
    LoadingTitle = "DarkMoon X",               -- Text displayed during loading process
    LoadingSubtitle = "Loading...",            -- Additional loading message
    
    ConfigSettings = {
        RootFolder = nil,                       -- Custom root directory (nil for default)
        ConfigFolder = "ScriptFolderName"       -- Folder name for storing configurations
    },
    
    KeySystem = false,                          -- Enable/disable authentication system
    KeySettings = {
        Title = "Hub Name",                     -- Key system window title
        Subtitle = "Key System",               -- Key system window subtitle
        Note = "Enter the Key Obtained for the Hub Name",  -- Instructions for users
        SaveInRoot = false,                     -- Save key in root folder instead of workspace
        SaveKey = true,                        -- Remember the key for future sessions
        Key = {"YourKey"},                     -- Array of valid authentication keys
        SecondAction = {
            Enabled = false,                    -- Enable secondary action button
            Type = "Discord",                   -- Action type: "Discord", "Website", etc.
            Parameter = "YourDiscordInvite"     -- Discord invite code or website URL
        }
    }
})

Window Configuration Parameters

Parameter Data Type Default Value Description
Name string Required Primary window title shown in the header bar
Subtitle string "" Secondary descriptive text beneath the title
LogoID string "" Roblox asset ID for custom logo display
Size UDim2 UDim2.new(0.6, 0, 0.7, 0) Window dimensions as screen percentage
LoadingEnabled boolean false Controls visibility of loading screen

Notification System

Create informative notifications for user feedback:

local Notify = NewNotify({
    Title = "Success!",                        -- Bold notification header text
    Description = "Operation completed successfully.",  -- Detailed message content
    Time = 5                                   -- Display duration in seconds
})

Notification Parameters

Parameter Data Type Range Description
Title string 1-50 chars Main notification header
Description string 1-200 chars Detailed message body
Time number 1-30 seconds Auto-dismiss timer duration

User Interface Components

Tab Management

Create organized navigation tabs for your interface:

local MainTab = Window:CreateTab({
    Name = "Main",                             -- Tab display name in navigation
    Icon = "home",                            -- Icon identifier (Material icons or asset ID)
    ImageSource = "Material",                  -- Icon source: "Material" for built-in, "Custom" for asset ID
    ShowTitle = true                          -- Display tab name alongside icon
})

Tab Configuration Options

Parameter Data Type Options Description
Name string Any text Displayed tab label
Icon string Material names or asset ID Visual tab identifier
ImageSource string "Material", "Custom" Icon rendering method
ShowTitle boolean true, false Text visibility control

Material Icon Examples: "home", "settings", "user", "shield", "star", "tools", "info", "help"

Content Organization

Section Dividers

Create visual separators within tabs:

MainTab:CreateSection("Player Options")        -- Creates a labeled section divider

Text Components

Status Labels

Display informational text with different styling:

MainTab:CreateLabel({
    Text = "Welcome to DarkLib V5!",           -- Content to display
    Style = 1                                  -- Visual style variant
})

Style Options:

  • 1 - Standard message (neutral blue)
  • 2 - Success message (green accent)
  • 3 - Warning/error message (red accent)

Rich Text Paragraphs

Create detailed text blocks with titles:

MainTab:CreateParagraph({
    Title = "About This Script",               -- Bold header text
    Text = "This is a comprehensive script hub with various features and tools for enhancing your gameplay experience."  -- Main content body
})

Interactive Elements

Action Buttons

Create clickable elements that execute functions:

local ExampleButton = MainTab:CreateButton({
    Name = "Execute Function",                 -- Button label text
    Description = "Runs the main function",   -- Optional tooltip description
    Callback = function()
        -- Function executed when button is clicked
        print("Button clicked!")
        -- Add your custom logic here
    end
})

Toggle Switches

Binary on/off controls with state persistence:

local ExampleToggle = MainTab:CreateToggle({
    Name = "Auto Farm",                        -- Toggle display name
    Description = "Automatically farms resources",  -- Feature explanation
    CurrentValue = false,                      -- Initial state (true = on, false = off)
    Callback = function(Value)
        -- Function called when toggle state changes
        print("Toggle state:", Value)          -- Value is true when on, false when off
        if Value then
            -- Code to execute when toggled ON
        else
            -- Code to execute when toggled OFF
        end
    end
}, "AutoFarmFlag")                            -- Unique identifier for config saving

Text Input Fields

User text input with validation options:

local TextInput = MainTab:CreateInput({
    Name = "Player Name",                      -- Input field label
    Description = "Enter target player name", -- Purpose explanation
    PlaceholderText = "Username...",          -- Hint text shown when empty
    CurrentValue = "",                        -- Pre-filled default value
    Numeric = false,                          -- true = numbers only, false = any text
    MaxCharacters = 20,                       -- Maximum input length limit
    Enter = true,                            -- true = require Enter key, false = instant
    Callback = function(Text)
        -- Function called when input is submitted
        print("Input received:", Text)
        -- Validate and process the input
        if Text ~= "" then
            -- Handle non-empty input
        end
    end
}, "PlayerNameFlag")                         -- Configuration save identifier

Numeric Sliders

Adjustable value controls with visual feedback:

local SpeedSlider = MainTab:CreateSlider({
    Name = "Walk Speed",                       -- Slider label
    Range = {16, 100},                        -- Minimum and maximum values
    Increment = 1,                            -- Step size for value changes
    CurrentValue = 16,                        -- Initial slider position
    Callback = function(Value)
        -- Function called when slider value changes
        print("New speed value:", Value)
        game.Players.LocalPlayer.Character.Humanoid.WalkSpeed = Value
    end
}, "WalkSpeedFlag")                          -- Configuration identifier

Selection Dropdowns

Multi-option selection menus:

local WeaponDropdown = MainTab:CreateDropdown({
    Name = "Select Weapon",                    -- Dropdown label
    Description = "Choose your preferred weapon",  -- Usage explanation
    Options = {"Sword", "Bow", "Staff", "Dagger"},  -- Available choices
    CurrentOption = {"Sword"},                 -- Default selected option(s)
    MultipleOptions = false,                   -- true = multi-select, false = single select
    Callback = function(Option)
        -- Function called when selection changes
        print("Selected weapon:", Option)
        if MultipleOptions then
            -- Option is a table when multiple selection is enabled
            for _, weapon in pairs(Option) do
                print("Selected:", weapon)
            end
        else
            -- Option is a string when single selection
        end
    end
}, "WeaponFlag")                             -- Configuration save key

Complete API Reference

Core Library Functions

Function Signature Return Type Purpose
DarkLib:CreateWindow(config) WindowObject Initialize main interface window
NewNotify(config) NotificationObject Display temporary user notification

Window Object Methods

Method Parameters Return Type Description
Window:CreateTab(config) Table TabObject Add new navigation tab
Window:CreateHomeTab(config) Table TabObject Generate information homepage

Tab Object Methods

Method Parameters Return Type Description
Tab:CreateSection(name) String nil Add visual section divider
Tab:CreateButton(config) Table ButtonObject Create interactive button
Tab:CreateToggle(config, flag) Table, String ToggleObject Add binary switch control
Tab:CreateInput(config, flag) Table, String InputObject Insert text input field
Tab:CreateSlider(config, flag) Table, String SliderObject Add numeric value slider
Tab:CreateDropdown(config, flag) Table, String DropdownObject Create selection menu
Tab:CreateLabel(config) Table LabelObject Display static text
Tab:CreateParagraph(config) Table ParagraphObject Show formatted text block
Tab:BuildConfigSection() None nil Add configuration controls
Tab:BuildThemeSection() None nil Add theme customization

Best Practices

Code Organization

Modular Structure: Organize related functionality into separate tabs and use descriptive section names to create logical groupings.

Consistent Naming: Use clear, descriptive names for all UI elements. Avoid abbreviations that might confuse users.

Flag Management: Create a centralized naming convention for configuration flags to prevent conflicts and ensure consistency.

User Experience Design

Progressive Disclosure: Present the most important features prominently while keeping advanced options in clearly marked sections.

Feedback Systems: Always provide immediate visual or textual feedback when users interact with controls.

Default Values: Set sensible default values that work for most users while still allowing customization.

Performance Considerations

Event Handling: Use debouncing for frequently triggered events like slider changes to prevent performance issues.

Resource Management: Store references to frequently accessed UI elements to avoid repeated object lookups.

Memory Optimization: Clean up unused callbacks and references when recreating UI elements.

Error Prevention

Input Validation: Implement proper validation for all user inputs, especially numeric fields and text inputs.

Range Checking: Ensure slider values and numeric inputs stay within acceptable ranges.

Null Checking: Always verify that required parameters are provided before executing callback functions.

Configuration Best Practices

Meaningful Flags: Use descriptive, unique flag names that clearly indicate their purpose.

Default States: Set appropriate default values for toggles and inputs based on the most common use case.

Persistence Strategy: Decide which settings should persist between sessions and which should reset.

Complex Integration Example

-- Complete example integrating multiple components
local GameplayTab = Window:CreateTab({
    Name = "Gameplay",
    Icon = "gamepad",
    ImageSource = "Material",
    ShowTitle = true
})

-- Player modifications section
GameplayTab:CreateSection("Player Modifications")

-- Speed control with validation
local speedEnabled = false
local currentSpeed = 16

local speedToggle = GameplayTab:CreateToggle({
    Name = "Custom Speed",
    Description = "Enable custom walk speed",
    CurrentValue = false,
    Callback = function(Value)
        speedEnabled = Value
        if Value then
            game.Players.LocalPlayer.Character.Humanoid.WalkSpeed = currentSpeed
        else
            game.Players.LocalPlayer.Character.Humanoid.WalkSpeed = 16
        end
    end
}, "SpeedToggle")

local speedSlider = GameplayTab:CreateSlider({
    Name = "Speed Value",
    Range = {16, 500},
    Increment = 1,
    CurrentValue = 16,
    Callback = function(Value)
        currentSpeed = Value
        if speedEnabled then
            game.Players.LocalPlayer.Character.Humanoid.WalkSpeed = Value
        end
    end
}, "SpeedValue")

-- Teleportation system
GameplayTab:CreateSection("Teleportation")

local savedPositions = {}

local positionInput = GameplayTab:CreateInput({
    Name = "Position Name",
    Description = "Name for saved position",
    PlaceholderText = "Enter position name...",
    CurrentValue = "",
    Numeric = false,
    MaxCharacters = 30,
    Enter = true,
    Callback = function(Text)
        if Text ~= "" then
            local player = game.Players.LocalPlayer
            if player.Character and player.Character:FindFirstChild("HumanoidRootPart") then
                savedPositions[Text] = player.Character.HumanoidRootPart.Position
                print("Position saved:", Text)
            end
        end
    end
}, "PositionName")
